# Submission pre-flight checklist (v2)

Run at **readiness** (T-5 / midpoint) and **final** (T-1 / <=4h before).
UNKNOWN blocks. Nothing passes silently.

## 1. Matrix
- [ ] Matrix supplied with mandatory rows found.
- [ ] Every mandatory row Green on all five RAGs (compliance, evidence,
      drafting, review, submission).

## 2. Answers
- [ ] Every required answer supplied.
- [ ] Every answer within its limit.
- [ ] No unfilled `[EVIDENCE: ...]` placeholders.
- [ ] No unresolved B04 checker flags.
- [ ] Every claim has an evidence-ledger row with a real source.

## 3. Documents (conditional — check applicability per bid)
For each required document: present · in-date (expiry recorded) · acceptable
for this bid · source link. Any UNKNOWN blocks.
- [ ] Insurances (EL / PL / PI / cyber) — as required by this procurement
- [ ] Carbon Reduction Plan — if thresholds/applicability require
- [ ] Modern Slavery statement — if required
- [ ] Pricing document
- [ ] Declarations (conflict, exclusion grounds)
- [ ] Certifications (Cyber Essentials / ISO) — if required

## 4. EFS / FVRA gate (BLOCKING)
Clears only if one is true and confirmed for this route:
- [ ] No separate financial-standing stage (e.g. below-threshold under PA23), or
- [ ] Standalone position clears the stated threshold, or
- [ ] Arobs PCG in place, current, acceptable **at this stage of this procurement**.

## 5. Deadline (BLOCKING)
- [ ] Date **and** time **and** timezone confirmed (ISO with offset).
- [ ] Submission planned with buffer, not on the deadline.

## 6. Clarification handling (BLOCKING — the G-Cloud 15 lesson)
- [ ] Named owner + named backup for the whole evaluation period.
- [ ] Submission/clarification mailbox confirmed monitored (shared, not a
      personal/former-staff inbox).
- [ ] B06 register created and handover done.

## 7. Logistics
- [ ] Correct portal, correct lot(s), correct submission locations per matrix.
