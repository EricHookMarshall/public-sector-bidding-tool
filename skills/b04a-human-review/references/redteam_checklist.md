# Red-team checklist

Work through these per answer. Challenge, don't rubber-stamp.

## Score as the evaluator
- [ ] Does it directly answer *this* question, or an easier adjacent one?
- [ ] Would it earn the marks available? Where would it lose marks?
- [ ] Is the evaluation criterion's language mirrored so scoring is easy?
- [ ] Is the buyer benefit explicit, or only implied?

## Evidence challenge
- [ ] Is every claim backed by real, current, specific evidence?
- [ ] What is the single weakest claim? Cut or strengthen it.
- [ ] Any metric that can't be substantiated if challenged?
- [ ] Any expired/needs-update evidence still relied on?

## Compliance re-check
- [ ] Within the word/character limit?
- [ ] All parts of a multi-part requirement addressed?
- [ ] Correct submission location recorded?
- [ ] Regime-correct language (PA23/MAT, RM codes, GCA)?

## Risk & commitment
- [ ] Any "we will" commitment needing commercial/MD approval?
- [ ] Any named person/client without approval?
- [ ] Any implied liability/SLA to qualify?
- [ ] Any capability implied outside the Microsoft Practice envelope?

## Decision
- [ ] GO — clears to B05.
- [ ] REWORK — named actions, loops to B04.
